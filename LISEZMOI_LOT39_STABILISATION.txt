Pourquoi les réservations/prix se synchronisaient MAIS PAS les
sous-locations ni l'entretien :

  • Le serveur SAUVEGARDAIT la sous-location dans un champ « items ».
  • L'application allait la CHERCHER dans un champ « value ».
  • Résultat : la donnée arrivait bien, mais l'app lisait le mauvais
    champ → elle recevait « rien » → rien ne s'affichait.

Les réservations marchaient car elles utilisent « items » des deux
côtés. Les sous-locations/entretien utilisaient « value » côté app
mais « items » côté serveur → désaccord → pas de synchro.

✅ CORRIGÉ : le serveur stocke désormais « value », et reste
compatible avec les anciennes données déjà en base. Une fois ce
ZIP redéployé sur Cloudflare, les sous-locations ET l'entretien se
synchronisent comme les réservations, sur tous les appareils.

  ⚠ Ta configuration Cloudflare était CORRECTE depuis le début
  (KV ASL_DB + variables ADMIN_USER/ADMIN_PASS/ADMIN_KEY). Le
  problème était uniquement dans le code. Il faut juste redéployer
  ce ZIP.
